Kanonik SOP & POS

Aljabar Boolean: Sum of Products (SOP) & Product of Sums (POS)

1. Aljabar Boolean

Aljabar Boolean adalah sistem matematika untuk nilai biner (0 dan 1), dicetuskan George Boole (1854). Seluruh komputasi digital modern berpijak pada sistem ini.

Tiga operasi dasar

Operasi Simbol Nama Keterangan
AND · Konjungsi Bernilai 1 hanya jika semua input 1
OR + Disjungsi Bernilai 1 jika minimal satu input 1
NOT ¬ / ' Komplemen Membalik nilai: 0→1, 1→0
Fungsi Kanonik: ekspresi boolean di mana setiap suku memuat semua variabel tepat satu kali, baik normal maupun komplemen. Ada dua bentuk: kanonik SOP (berbasis minterm) dan kanonik POS (berbasis maxterm).

Untuk fungsi dengan n variabel, terdapat 2ⁿ kombinasi input yang mungkin. Setiap kombinasi memiliki satu minterm dan satu maxterm yang bersesuaian.

2. Minterm & Kanonik SOP

Minterm adalah blok pembangun kanonik SOP. Setiap minterm bernilai 1 hanya untuk satu kombinasi input tertentu.

Definisi Minterm: Minterm ke-i, dinotasikan mᵢ, adalah produk (AND) semua variabel di mana: bit = 1 ditulis normal, bit = 0 ditulis komplemen.
Aturan: Bit = 1 → tulis normal (A, B, C).   Bit = 0 → tulis komplemen (A', B', C').
-- Minterm untuk 3 variabel (A, B, C) --
m₀ = A'B'C' ← A=0,B=0,C=0
m₁ = A'B'C ← A=0,B=0,C=1
m₂ = A'BC' ← A=0,B=1,C=0
m₃ = A'BC ← A=0,B=1,C=1
m₄ = AB'C' ← A=1,B=0,C=0
m₅ = AB'C ← A=1,B=0,C=1
m₆ = ABC' ← A=1,B=1,C=0
m₇ = ABC ← A=1,B=1,C=1
Kanonik SOP: OR dari semua minterm di mana fungsi bernilai 1. Dinotasikan dengan Σ (sigma).
-- f bernilai 1 pada baris 1, 3, 5, 7 --
f(A,B,C) = Σ(1,3,5,7) = A'B'C + A'BC + AB'C + ABC
-- Setelah disederhanakan --
= C

Indeks A B C f Minterm

1 0 0 1 1 m₁ = A'B'C
2 0 1 0 0 —
3 0 1 1 1 m₃ = A'BC
4 1 0 0 0 —
5 1 0 1 1 m₅ = AB'C

7 1 1 1 1 m₇ = ABC

3. Maxterm & Kanonik POS

Maxterm adalah dual dari minterm. Setiap maxterm bernilai 0 untuk tepat satu kombinasi input.

Definisi Maxterm: Maxterm ke-i, dinotasikan Mᵢ, adalah jumlah (OR) semua variabel di mana: bit = 0 ditulis normal, bit = 1 ditulis komplemen.
Aturan (kebalikan minterm): Bit = 0 → tulis normal.   Bit = 1 → tulis komplemen.
-- Maxterm untuk 3 variabel (A, B, C) --
M₀ = (A + B + C) ← A=0,B=0,C=0
M₁ = (A + B + C') ← A=0,B=0,C=1
M₂ = (A + B'+ C) ← A=0,B=1,C=0
M₃ = (A + B'+ C') ← A=0,B=1,C=1
M₄ = (A'+ B + C) ← A=1,B=0,C=0
M₅ = (A'+ B + C') ← A=1,B=0,C=1
M₆ = (A'+ B'+ C) ← A=1,B=1,C=0
M₇ = (A'+ B'+ C') ← A=1,B=1,C=1
Kanonik POS: AND dari semua maxterm di mana fungsi bernilai 0. Dinotasikan dengan Π (pi).
-- f bernilai 0 pada baris 0, 2, 4, 6 --
f(A,B,C) = Π(0,2,4,6) = (A+B+C)(A+B'+C)(A'+B+C)(A'+B'+C)
-- Ekuivalen dengan SOP: Σ(1,3,5,7) = C --
Indeks A B C f Maxterm
0 0 0 0 0 M₀ = (A+B+C)
1 0 0 1 1 —
2 0 1 0 0 M₂ = (A+B'+C)
3 0 1 1 1 —
4 1 0 0 0 M₄ = (A'+B+C)
5 1 0 1 1 —
6 1 1 0 0 M₆ = (A'+B'+C)
7 1 1 1 1 —

4. Cara Konversi

Langkah sistematis mengubah tabel kebenaran menjadi ekspresi kanonik SOP atau POS.

1.

Buat tabel kebenaran

Tuliskan semua 2ⁿ kombinasi nilai variabel input beserta nilai output f.

2.

Tandai baris f = 1

Cari semua baris di mana f bernilai 1 dan catat nomor indeksnya.

3.

Tulis minterm tiap baris

Bit 1 adalah variabel normal dan bit 0 adalah variabel komplemen. Hasilnya adalah produk AND.

4.

Jumlahkan (OR) seluruh minterm

Hubungkan semua minterm dengan operator OR (+). Hasilnya adalah kanonik SOP.

5.

Tulis dalam notasi Σ

Ringkaskan: f(A,B,C) = Σ(daftar indeks), misalnya Σ(1,3,5,7).

-- Contoh: f(A,B,C) = Σ(0,2,6) --
Baris 0: A=0,B=0,C=0 → A'B'C'
Baris 2: A=0,B=1,C=0 → A'BC'
Baris 6: A=1,B=1,C=0 → ABC'
f = A'B'C' + A'BC' + ABC'
1.

Buat tabel kebenaran

Tuliskan semua 2ⁿ kombinasi nilai variabel input beserta nilai output f.

2.

Tandai baris f = 0

Cari semua baris di mana f bernilai 0 dan catat nomor indeksnya.

3.

Tulis maxterm tiap baris

Bit 0 → variabel normal; bit 1 → variabel komplemen. Hasilnya adalah jumlah OR.

4.

Kalikan (AND) seluruh maxterm

Hubungkan semua maxterm dengan operator AND (·). Hasilnya adalah kanonik POS.

5.

Tulis dalam notasi Π

Ringkaskan: f(A,B,C) = Π(daftar indeks), misalnya Π(1,4).

-- Contoh: f(A,B,C) = Π(1,4) --
Baris 1: A=0,B=0,C=1 → (A + B + C')
Baris 4: A=1,B=0,C=0 → (A'+ B + C)
f = (A+B+C') · (A'+B+C)

5. Perbandingan SOP vs POS

SOP dan POS adalah dua representasi ekuivalen untuk fungsi yang sama. Pilihan mana yang lebih efisien bergantung pada distribusi nilai 0 dan 1 dalam tabel kebenaran.

Σ   SOP — Sum of Products

  • Dibangun dari minterm (produk AND)
  • Menggunakan baris output = 1
  • Bit 1 = normal, bit 0 = komplemen
  • Suku-suku dihubungkan dengan OR (+)
  • Efisien bila jumlah nilai 1 lebih sedikit
  • Implementasi: AND kemudian OR

Π   POS — Product of Sums

  • Dibangun dari maxterm (jumlah OR)
  • Menggunakan baris output = 0
  • Bit 0 = normal, bit 1 = komplemen
  • Suku-suku dihubungkan dengan AND (·)
  • Efisien bila jumlah nilai 0 lebih sedikit
  • Implementasi: OR kemudian AND
